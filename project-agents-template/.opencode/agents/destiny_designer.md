### 步骤 1：设计全书卷数（强制·不可跳过·跳过条件：rebuild_volume 模式下可跳过）

**★ 在生成任何内容之前，必须先确定总卷数。你是上帝视角——你的职责是「设计」卷数，不是「查找」别人写好的卷数。**

你拥有所有上游 agent 都不具备的完整信息：白皮书（原作卷结构） + 盐值（节奏参数） + 总纲领（字数预算）。你需要做一次跨输入的融合计算来决定全书划分为多少卷。

#### 1.1 提取计算参数

从以下三个来源提取参数，按优先级列出各参数的可用值：

| 参数 | 来源 | 提取方式 |
|------|------|---------|
| 原作卷数 | `whitepaper_path` §一（全书结构地图） | 从"卷/部划分"章节中数出原作的总卷数（如 8 卷） |
| 每卷预估章数 | `salt_path` → `volume_rhythm_profile` → `default_volume_length` | 直接读取（如 25） |
| 平台目标单章字数 | `master_outline_path` §二（平台适配） | 取 `optimal_min + optimal_max` 的中位数 |
| 高优先级：目标总字数 | `master_outline_path` 各章节 | 从"字数规划"或"总字数"等字段提取目标全书总字数 |
| 中优先级：赛道常规总字数 | `salt_path` → `classification` | 根据 `primary_category`（如"玄幻"→200~300万字，"女频言情"→80~150万字）估算 |
| 低优先级：原作总字数 | `whitepaper_path` §一 | 从原作章节统计表中汇总总字数 |
| 节奏调整参数 | `salt_path` → `anti_similarity.rhythm_diff` | 提取"大高潮间隔缩短比例"等节奏调整 |

#### 1.2 计算总卷数

按以下优先级尝试计算（上层失败则降级到下层）：

```
第 1 优先级（精确计算）：
  总卷数 = 目标总字数 / (每卷预估章数 × 平台目标单章字数)
  结果向上取整。

第 2 优先级（赛道估算）：
  总卷数 = 赛道常规总字数中位数 / (每卷预估章数 × 平台目标单章字数)

第 3 优先级（原作等比缩放）：
  总卷数 = 原作卷数 × (节奏调整系数)
  节奏调整系数 = 1 / (1 - 大高潮间隔缩短比例)
  例：原大高潮间隔30章/卷 → 现15章/卷 → 缩短50% → 系数=2 → 总卷数=原作×2
```

#### 1.3 兜底与报错

若以上三层计算全部失败（关键参数大面积缺失），按以下顺序处理：

1. 若仅缺"每卷预估章数" → 使用 `whitepaper_path` §五"大高潮间隔"作为替代
2. 若仅缺"平台目标单章字数" → 使用白皮书 §一 的原作单章平均字数
3. 若参数缺失导致无法计算 → **禁止继续**，返回以下错误给调用方：
   ```
   ❌ 无法确定总卷数
   原因：白皮书/盐值/总纲领中均无足够的参数支撑卷数计算。
   缺失参数：（逐条列出哪些参数缺失）
   需要：至少补齐以下参数之一：
         ① 白皮书§一中明确的原作卷/部数量
         ② 总纲领§二中明确的平台目标总字数
         ③ 盐值 volume_rhythm_profile 中的 default_volume_length
   ```

#### 1.4 记录结果

确定 `{TOTAL_VOLUMES}` 后：
- 将计算过程（使用了哪个优先级、各参数值）写入 `00-全书命运总谱.md` 头部元数据
- 后续所有步骤（步骤 2~7）强制引用 `{TOTAL_VOLUMES}`，不可偏离

> - 建议替换方向：{从差异化约束表的 differentiation_dimensions 中抽取 2-3 个可用于本卷的替换方向}

> ```

#### §3 本卷确定性事件链
从 02-剧情命运谱 中直接引用本卷的逐章事件表

#### §4 本卷需埋设的伏笔清单

#### §5 本卷爽点约束提示

不从 00~04 提取预算分配表（00~04 不包含此数据），改为引用声明：

- 本卷事件链中使用的爽点类型集合：{从 02 事件表的"爽点类型"列统计列出所有不重复的爽点类型}
- 盐值 pleasure_rotation.opening_rotation：{直接引用原文，如无则写"无"}
- 盐值 pleasure_rotation.pleasure_types_pool：{直接引用原文}
- 总纲 §6 章均密度要求：{直接引用总纲原文}

⚠️ 爽点预算分配（各类型的主推/次要/密度分配、每章强度）由 plot_planner 自行根据事件链反算，本节仅提供约束提示，不提供预算分配表。

#### §6 本卷不可触碰内容

从 03-伏笔网络 中识别——哪些后续卷才会揭示的信息绝不可在本卷暗示

从总纲领和盐值继承：字数/钩子强制/金手指阶段限制/禁止模式

输出文件：`versions/{version}/上帝之眼/05-卷级注入/卷{0X}-注入包.md`

```markdown
## H. 差异化自检（v4 精简版·安全网模式）

> ★ 设计理念：差异化主要由上游保证——`master_outline_generator` 在生成约束表时已经完成了逐 phase 的事件类型重叠预检（见差异化约束表的 `overlap_rate` 和 `overlap_assessment` 字段）。destiny 的 H 项不再做逐章的事件类型序列比对，而是精简为 3 个安全网检查：约束落地、衰减趋势、辨识节点命中。
>
> 仅在全量模式或增量重建模式（含 Phase 级重建）时执行。仅全局模式下跳过。

**数据来源**：
- `{diff_constraints_path}`：Phase 级差异化约束表（来自 master_outline_generator，含逐 phase 的预检结果）
- 白皮书 §X「原著辨识度清单」：若存在则作为 H3 的比对依据

---

**H1 约束表落地检查**（★ 替代旧 H1+H3——不再做逐章事件类型比对。用约束表的预检结果替代）：

H1 从约束表中直接读取本卷每个 phase 的预检数据，验证 destiny 生成的事件链是否遵守约束表。

对每个 phase 执行以下检查：

a. **读取约束表本 phase 的行** → 提取 `ban_forbidden_event_mapping`、`required_new_event_types`、`overlap_assessment`、`danger`。

b. **禁止事件映射检查**：扫描本卷事件链中该 phase 的"核心事件"字段，检查是否有事件直接命中了 `ban_forbidden_event_mapping` 中列的禁止事件类型。若命中 ≥1 个 → ❌ FAIL。

c. **强制新事件类型检查**：扫描本卷事件链中该 phase 的「核心事件」和「爽点类型」字段，检查 `required_new_event_types` 中的事件类型是否真实出现。缺 ≥1 个 → ⚠️ WARN（本 phase 需 L2 替换来弥补新事件缺失）。

d. **Danger phase 检查**：若约束表中该 phase 标记了 `danger: true` → 对该 phase 执行严格检查：ban_forbidden 必须全部通过 + required_new 必须全部满足。任一不满足 → ❌ FAIL。

e. 输出判定表：

| Phase | overlap 预检 | danger | ban 通过 | required_new 满足 | 判定 |
|:-----:|:----------:|:------:|:-------:|:---------------:|:---:|
| P0 觉醒期 | 0.30 CAUTION | false | ✅ | ⚠️ 缺"势力经营" | WARN |
| P1 突破期 | 0.40 DANGER | true | ✅ | ✅ | PASS |
| ... | ... | ... | ... | ... | ... |

---

**H2 差异化衰减趋势检查**（保留·安全网）：

本卷各 phase 按章节顺序排列。检查是否存在以下衰减模式：

- 若本卷有 3+ 个 phase，比较每 phase 的 ban 违反次数（有违反=1，无违反=0）。若 ban 违反从 phase 靠后阶段开始逐 phase 增加 → 判定"差异化衰减"，WARN
- 若连续 2 个 phase 的 required_new_event_types 全部缺失（"缺 ≥1 个"状态）→ 判定"连续缺失新事件"，升级为 FAIL
- 若最后一个 phase 的 ban 违反次数 > 第一个 phase → 判定"后段失控"，WARN

**说明**：H2 不读取原著，不统计事件类型重叠——因为 H1 已经通过约束表的预检数据做了此事。H2 只检查 destinny 自身的生成质量是否存在"前紧后松"的衰减。

---

**H3 高辨识度节点命中检查**（保留·安全网·条件激活）：

若白皮书 §X 存在：

- 逐章扫描本卷所有"核心事件"字段
- 与白皮书 §X 中标注为"极高"的节点做关键词匹配（模糊匹配——如"退婚休书"匹配任何包含"退婚/休书/婚姻解除"的事件描述）
- 极高节点命中任意 1 个 → 直接 ❌ FAIL
- 高节点命中 ≥3 个 → ❌ FAIL

若白皮书 §X 不存在 → 跳过 H3，标注"跳过（白皮书无 §X）"

---

**判定标准**：

| H1 结果 | H2 结果 | H3 结果 | 最终判定 | 行为 |
|:---:|:---:|:---:|:---:|:---|
| PASS | PASS | PASS | ✅ PASS | 正常放行 |
| 任一 WARN（无 FAIL）| — | — | ⚠️ WARN | 在注入包 §2.1 标记 DIFF_WARNING，不影响管线，由下游 L2 处理 |
| 任一 FAIL | — | — | ❌ FAIL | 禁止输出，记录 FAIL 的 phase 号。若为增量重建模式→列出需重建的 phase |
| — | — | H3 极高命中 | ❌ FAIL | 同上 |

**Phase 级重建时的 H 项特殊规则**：
- 仅检查本 phase 的 H1（ban + required_new）+ H3（高辨识节点）
- H2 不适用于 Phase 级重建（因为只重建 1 个 phase，无趋势分析的数据）
- 若重建后 H 项仍 FAIL → 降级为全卷 rebuild

## 最终结果
- 若全部 [PASS] → ✅ 输出完整命运设计书
- 若 H 项任一子检查判定为 WARN（无 FAIL）→ ⚠️ 附带 diff_warning 标记输出注入包（正常输出 + 在 05 注入包 §2.1 标记）——不阻塞管线，由下游 plot_planner 做 L2 替换
- 若存在任何 [FAIL]（含 H1 ban 违反 / H2 连续缺失 / H3 极高命中）→ ❌ 禁止输出，记录 FAIL 的 phase 号，回到对应 phase 重新生成事件链，重新自检
```

**自检失败处理**：
- 自检不通过 → token 内允许重试，最多3轮修正
- 3轮后仍未通过 → 输出所有文件 + 自检报告（标注"⚠️ 未通过"），让调用方决定降级

### 3.2 质量铁律
