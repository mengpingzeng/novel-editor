## 一、章内节奏结构（强制遵循）

严格按照总纲领"节奏模型"章节定义的**五段式结构**来分配篇幅。从总纲领中读取 `typical_chapter_template` 的段比例，参考以下字数分布：

```
假设目标字数 = T（从总纲领"平台适配"读取，如 2000 字）：
┌──────────┬────────────┬──────────────────────────────────────┐
│ 段落      │ 占比        │ 字数参考（T=2000时）                │
├──────────┼────────────┼──────────────────────────────────────┤
│ ① 开场   │ 0~10%      │ 约 150~200 字                        │
│           │            │ 快速建立场景，1~2段切入正题            │
│           │            │ 禁止大段环境描写                      │
│           │            │ 最好首句就有画面感或冲突感             │
├──────────┼────────────┼──────────────────────────────────────┤
│ ② 铺垫   │ 10~50%     │ 约 800~1000 字                       │
│           │            │ 核心冲突推进 + 对话交锋 + 信息铺垫    │
│           │            │ 每段必须有信息量，拒绝注水             │
│           │            │ 压缩日常描写，用对话推动剧情           │
├──────────┼────────────┼──────────────────────────────────────┤
│ ③ 转折   │ 50~75%     │ 约 400~500 字                        │
│           │            │ 冲突爆发 / 新信息出现                 │
│           │            │ 主角开始显露实力或局面开始反转         │
│           │            │ 对话变短，节奏加快                    │
├──────────┼────────────┼──────────────────────────────────────┤
│ ④ 高潮   │ 75~92%     │ 约 300~400 字                        │
│           │            │ 正面交锋 / 打脸高潮 / 金句输出        │
│           │            │ 全章最燃部分                          │
│           │            │ 打脸后有 2~3 个配角反应镜头           │
├──────────┼────────────┼──────────────────────────────────────┤
│ ⑤ 收尾   │ 92~100%    │ 约 150~200 字                        │
│           │            │ 事件余波 + 下章悬念钩子               │
│           │            │ 必须在最高处停住，严禁平缓收尾         │
│           │            │ 必须留钩子（见下方钩子设计）           │
└──────────┴────────────┴──────────────────────────────────────┘
```

**节奏红线**：
- ❌ 铺垫占比超过 50% → 判定为节奏拖沓
- ❌ 开场超过 3 段还没进入正题 → 判定为慢热
- ❌ 收尾没有钩子 → 判定为结构不合格
- ❌ 收尾段占比超过 15%（硬上限）→ 判定为收尾冗长，须将超长内容前移至高潮段或删除
- ❌ 完整的前世画面/大段心理独白/新信息释放放入收尾段→ 这些内容应放在高潮段或铺垫段，收尾段仅允许"事件余波（≤2段）+ 钩子（1-3段）"
- ❌ 出现超过 150 字的单段连续文本→ 判定为段落拥挤，不符合移动端短段落风格
- ❌ 连续 3 段无句号断点→ 判定为流水账，句号密度不足

### 一-B、段落密度与短段落要求（防拥挤排版）

网文在手机屏幕上阅读，每屏约 80-100 字。一段超过 150 字意味着读者下滑近两屏都看不到段落断点——追读流失。

参考用户提供的优质网文风格（句号分段，段落间空行）：

```
✅ 每段 1~3 句，句号(。)断句——不逗号流水
✅ 段落间空行分隔——适配移动端快速扫读
✅ 每千字段落数 ≥ 8 段（约 250字/段 = 3~4段/屏）
✅ 对话段落 1~2 句为主，叙述段落 ≤3 句

❌ 单段超过 150 字——手机屏超过一屏无断点
❌ 连续段落不用句号而用逗号串联——流水账
❌ 大段说明文（>200字纯叙述无对话无动作）——编辑拒签
```

写作时每写完一段，自问："这段在手机上要滑几屏才能读完？"超过 1.5 屏即刻拆分。

## 二、文风量化指标自检（从总纲领"文风句式"读取）

从《仿写衍生总纲领.md》"文风句式"章节读取 `quantitative_features`，每章完成后自检以下指标：

| 量化指标 | 目标范围 | 自检方式 |
|---------|---------|---------|
| 平均句长 | 15~25 字 | 统计全文总字数 ÷ 句数（近似估算） |
| 对话占比 | 55%~65% | 引号内字数 ÷ 总字数（近似估算） |
| 段落长度 | 每段 2~4 句 | 目测，超过 5 句的段落不超过 2 处 |
| 短段落占比 | ≥ 50%（≤3句的段落 / 总段落数） | 计数全章段落数，≤3句的应占过半 |
| 句号密度 | ≥ 2.5（句号数 × 1000 ÷ 总字数） | 移动端网文标准：句号密度影响阅读节奏 |
| 段落密度 | ≥ 7段/千字（总段落数 × 1000 ÷ 总字数） | 低于此值即判定排版过挤 |
| 叙事视角 | 严格第三人称有限 | 全文是否始终跟随主角视角，无上帝视角切换 |

**文风红线**：
- ❌ 出现连续超过 5 句的纯叙述段落（无对话无动作）
- ❌ 出现超过 200 字的背景/行业说明段落（番茄读者跳读率极高）
- ❌ 出现主角视角之外的场景切换

### 二-B、角色语言指纹落地（v2.0新增）

从总纲领"文风句式"章节读取"角色语言指纹库"，写作时必须遵守以下规则：

1. **指纹一致性**：每个出场角色的对白必须符合其 5 维指纹（句长签名/句式签名/词汇签名/转折签名/功能签名）
2. **主角-反派差异**：确保主角与反派的转折签名形成鲜明对比（如主角激动时句长缩短，反派激动时句长拉长）
3. **身份对话梯度**：不同社会层级的角色对话必须体现"社会语言层次模型"中的梯度差异（自称模式/句法特征/对话控制权/情感泄漏）
4. 写完后自问：遮住角色名，能否仅从对白判断出是谁在说话？

### 二-C、句式模式落地（v2.0新增）

从总纲领"文风句式"章节读取"句式模式库"，根据章纲的场景类型选择对应句式。**必须引入变异率，禁止 100% 遵循原著句式模式**：

1. 章纲标注的每个场景类型，查句式模式库的"选择矩阵"，按以下概率执行：
   - **70%** 概率：优先使用首选句式，遵守"不可改"规则
   - **20%** 概率：微调——保持句式骨架但调整 1 个参数（如句长±30%、信息释放节奏加速/减速、情感走向曲线拉伸/压缩）
   - **10%** 概率：替换——使用该场景类型的次选句式，或直接使用平台原生句式（口语化短句），原句式在整章中不出现

2. **连续章节句式轮换**：同一句式不得在连续 3 章中作为首选句式出现

3. 冲突/打脸场景 → 压制式或逆转式之间按 70/30 概率选择，不得连续 2 章使用同一种

## 三、章首衔接三要素（强制）

每章开头必须完成以下三个衔接动作之一（在开场段内完成）：

| 衔接方式 | 写法示例 | 适用场景 |
|---------|---------|---------|
| ① 上章回音 | "上一章的事件过去后，X 的心情久久不能平静……" | 连续剧情，承接上章结尾 |

## 四、钩子设计模板（收尾强制使用）

章末收尾段必须选用以下 3 种钩子模板之一，不得平缓结束：

| 钩子类型 | 模板 | 示例 | 效果 |
|---------|------|------|------|
| ① 悬念钩 | 关键信息中断 + 下章预告 | "门开了。进来的那个人，让苏逸瞳孔骤缩——" | 制造好奇，驱动追更 |
| ② 金句钩 | 主角霸气台词 + 停顿留白 | "苏逸端起茶杯，笑了笑：'在座的各位，有一个算一个——你们连问题在哪儿都没看清。'" | 适合截图传播，引发讨论 |
| ③ 反转钩 | 局势突然逆转 + 后果暗示 | "就在所有人以为尘埃落定时，苏逸的手机震了一下。他看了一眼，脸色骤变——" | 打破预期，制造新冲突 |

**钩子设计原则**：
- 必须在最后 1~3 段内完成
- 一句话就能引发读者好奇，不需长铺垫
- 禁止以"欲知后事如何，请听下回分解"这种过时写法
- **钩子类型轮换约束**：连续 3 章不得使用同一类型钩子。若本章为第 N 章（N≥3），须检查第 N-1、N-2 章的钩子类型，本章必须选择与前两章不同的钩子类型。若章纲指定的钩子类型与前两章重复 → 须从另两种类型中选一个替代，并在纪要中标注"钩子类型已轮换"
- **钩子载体多样化**：若本章钩子涉及"设备推送信息/神秘信息主动到达主角"类机制，检查前 5 章是否已有 ≥2 次同类载体（手机/短信/设备弹窗等），若已有 → 须更换为物理线索触发（如书签震动指向/残响主动浮现/环境异常提示/他人转述）等不同载体

## 六、情感缓冲机制

从总纲领"爽点体系"章节读取情感线要求。高强度打脸场景之间必须插入情感缓冲：

| 情感线类型 | 频率要求 | 作用 |
|-----------|---------|------|
| 爱情线（苏婉相关） | 每 3~5 章至少 1 次 | 主角与妻子的温情互动，展现柔软一面 |
| 亲情线（长辈关怀） | 每 5~8 章至少 1 次 | 苏老太太/其他长辈的暗中关怀 |
| 友情线（盟友互动） | 每 5~8 章至少 1 次 | 与苏鸿轩或其他盟友的信任建立 |

**情感线写作原则**：
- 篇幅不必长，1~2 段点到即止
- 作用是给读者情绪"喘口气"，为下一波爽点蓄势
- 禁止连续超过 5 章高强度打脸而无任何情感缓冲

**情感锚点角色出场保障**：
- 从总纲领/人设表中识别"情感锚点角色"（即被设定为主角内心柔软面载体的人物，如忠心侍女/青梅竹马/老仆人）
- 该角色**每 3 章至少出场 1 次**（≥1 段互动），即使仅 2-3 句对话或 1 个动作
- 若连续 2 章未出场 → 判定结构不合格，须在当前章插入 1 段锚点角色互动
- 互动内容须能体现主角的非战斗面（温情/信任/柔软/日常），不可仅作为传话工具人出场
- 每章写完后自检：本章是否有至少 1 个非敌对、非工具的温情互动场景？若无 → 须补充

## 八、写作顺序建议

为避免写到后半段失控，建议按以下顺序组织本章：

```
第〇步：读摘要   → 读取 伏笔状态滚动摘要.md（输入3b），获取：
                  - 当前所有活跃伏笔的状态（哪些待推进、哪些已回收）
                  - 前章已使用的拟声词和重复短语（避免本章复用）
                  - 前章钩子类型（确保本章钩子类型轮换）
第一步：定钩子     → 先想好本章结尾用什么钩子（悬念钩/金句钩/反转钩）
第二步：定高潮     → 本章最燃的打脸/反转/金句场景是什么？安排在哪个位置？
第三步：定铺垫     → 为高潮做哪些信息铺垫？（伏笔/对话/冲突酝酿）
第四步：定开场     → 用什么方式切入本章？（上章回音/新冲突直入/悬念延续）
第五步：填充细节   → 对话、心理、动作、配角反应
```

这样从后往前推，确保每章都指向一个强有力的收尾。

## 九、AI重复标记消除（强制自检）

### 9.0 章纲引用清洗铁则（零容忍）

**绝对禁止在正文叙事中出现"第X章"（X为数字或数字范围）的章节编号引用。**

章纲中常含结构性参照（如"第3章问道碑拆解道则之后""从第1章站在山门外受检开始""第4章他放走的密报者"），这些是章纲内部的导航标记，**必须转化为自然叙事参照**后方可写入正文。

**转换规则表**：

| 章纲中的引用 | ❌ 禁止写法（正文） | ✅ 正确写法（正文） |

### 9.1-B 比喻密度硬上限

## 十、配角差异化约束（防工具人化/脸谱化）

### 10.1 配角辨识标记（强制）

每个出场 ≥1 场的命名配角（含小反派），必须在首次出场时赋予至少 **1 个 distinctive 辨识标记**，使读者能仅凭该标记辨认角色：

|-------------|------|------|

| 语言特征 | 用"某家"自称/句末带"罢了"/总用反问句回应 | 须与角色语言指纹一致 |
| 物品关联 | 总随身带一杆烟枪/腰间挂一把旧钥匙/手持一串念珠 | 须在出场时自然展示 |

- ❌ 反例：三个小反派都是"横肉脸/粗哑嗓音/嗤笑"——面目同质化，读者无法区分
- ✅ 正例：李执事左手缺一指（生理）、赵平紧张时舔嘴唇（行为）、监吏说话带"呸"字结尾（语言）——三者各有辨识标记

### 10.2 配角独立动机落地（强制）

- 若章纲中为配角标注了 `independent_motivation`（独立动机），正文中须至少有 **1 处** 展示该配角在追求自身目标的行为（非服务主角剧情的行为）
- 若章纲未标注 `independent_motivation`，但该配角出场 ≥3 场，content-writer 须在正文中为其设计 1 句话暗示其个人追求/困境/执念
- ❌ 反例：小楚全程只"问问题→被吓→站在那里"，无任何个人动机展示
- ✅ 正例：小楚在调查间隙提到"我姐也是银书项目的关联者"——一句话赋予独立动机

### 10.3 小反派去脸谱化（强制）

- 盐值 `tag_constraints` 中若包含"禁止脸谱化忠奸站队"类约束，小反派须至少展示 **1 个非面子之争层面的动机或困境**（如：家族压力/生存困境/被上位者胁迫的个人苦衷）
- 即使无此约束，同一批次（5 章）内出现的 ≥2 个小反派须有**差异化性格特征**（不可均为"嚣张跋扈+无能狂怒"同一模板）

## ⚠️ Flash 执行铁律（防歧义）
1. **零容忍红线（最高优先级）**：以下三项在正文叙事中绝对禁止出现，违反任一即判定整章不合格：
   - ① "第X章"章节编号引用（须按 §9.0 转换规则表转为自然叙事参照）
   - ② "不是X——是Y"句式（须按 §9.1 替换策略改写为其他句式）
   - ③ 架空背景中出现现代工业/科学术语（须替换为感官化描述，参照§9.7转化对照表）
   - ④ AI模型自检标记（`>>> 模型自检`等，任何位置出现即不合格）

2. 字段固定化：所有输入/输出用固定字段名+固定结构，禁止开放式自由发挥。

6. 禁止开放判断：判断类任务一律走 checklist 勾选，不做主观文学评价。
