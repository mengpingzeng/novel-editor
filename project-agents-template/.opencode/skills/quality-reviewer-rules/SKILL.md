## 零、强制 Grep 扫描清单

审核正文前必须先执行以下 12 项扫描。未完成即评分 → 审核无效。

| 序号 | 扫描项 | Grep 正则模式 | 命中阈值 | 命中后标记 |
|------|--------|-------------|---------|-----------|
| G1 | 章纲引用残留 | `第\d+章` | **0处（零容忍）** | "章纲引用残留" |
| G2 | "不是X——是Y"句式 | `不是.{1,15}[—–-]是` | **0处（零容忍）** | D2/D9 扣分 |
| G3 | 生产元数据残留 | `>>>.*自检\|【自检】\|字数预估\|deepseek\|glm.*ioa\|忠于章纲\|标签覆盖` | **0处** | "元数据残留" |
| G4 | 反应场景模板化 | `全场.{0,4}(寂静\|鸦雀\|安静\|死寂)\|满堂.{0,4}(寂静\|鸦雀\|安静\|死寂)\|满殿.{0,4}(寂静\|死寂)` | **0处** | D2 扣分 |
| G5 | 原著高频词未变异 | `瞳孔一缩\|倒吸凉气\|场中寂静` | **≤1处** | 超标 → D2 扣分 |
| G6 | 现代术语泄露 | `HRC\|布氏\|洛氏\|DNA\|蛋白质\|百分比` | **0处** | D8 扣分 |
| G7 | 标签元数据泄漏 | `标签[:：]` | **0处（零容忍）** | "标签元数据泄漏" |
| G8 | AI模型自检标记残留 | `>>>.*自检\|>>>.*ioa\|>>>.*flash\|>>>.*glm\|>>>.*deepseek\|模型自检` | **0处** | "AI标记残留" |
| G9 | 盐值fingerprint禁用词 | 从 project_salt.json 读取 fingerprint_variation_table，逐词扫描 | 不超过 max_per_chapter | D2 扣分 |
| G10 | 新重复短语检测 | 扫描 ≥4字短语，统计出现次数 | 同短语 ≤3次/章，≤5次/本卷（卷切换归零，从注入包 §1 读取本卷章号范围；跨卷检查 best-effort，超时 30s 跳过不扣分） | D2 扣分 |
| G11 | 章节标题Markdown残留 | `^#\s+第` 或 `^#+\s` | **0处** | "Markdown残留" |
| G12 | 对话标点符号错误 | `"`（ASCII双引号 U+0022）\| `「\|」`（U+300C/D）\| `『\|』`（U+300E/F） | **0处（零容忍）** | **整篇0分** "对话标点错误-需重写" |

扫描结果报告格式（审核输出开头必须呈现）：
```
【Grep工具扫描结果】
- G1 章纲引用残留：命中 X 处（行号：___）
- G2 "不是X——是Y"句式：命中 X 处
- ...
- G12 对话标点符号错误：命中 X 处（行号：___）【命中即整篇0分】
```

## 一、审核维度（100 分制）

### D1：字数合规（权重 5 分）

| 阈值 | 值 |
|------|-----|
| 绝对下限（低于 → 整篇 0 分） | 1000 字 |
| 满分下限 | 1500 字 |
| 满分上限 | 2000 字 |
| 绝对上限（超过 → 整篇 0 分） | 2500 字 |

得分规则：

| 实际字数 | D1 得分 |
|----------|---------|
| < 1000 | 整篇 0 分 |
| [1000, 1500) | `5 × (字数 - 1000) / 500` |
| [1500, 2000] | **5（满分）** |
| (2000, 2500] | `5 × (2500 - 字数) / 500` |
| > 2500 | 整篇 0 分 |

执行顺序（最高优先级）：
1. 先运行 G12 扫描对话标点，若命中 → 立即终止，总分 0
2. 统计本章实际字数
3. 若 < 1000 或 > 2500 → 立即终止，总分 0
4. 若在过渡区间 → 按公式计算 D1，继续审核 D2-D10
5. 若在满分区间 → D1 = 5，继续审核 D2-D10

---

### D2：文风仿写（权重 6 分）

| 检查项 | 扣分规则 |
|-------|---------|
| 平均句长 15~25 字 | 偏离每 5 字 -3 |
| 段落长度 2~4 句为主 | 超过 5 句的段落每处 -1，最多扣 4 |
| 视角一致性 | 出现非主角视角 -4 |
| 大段说明文 | 超过 200 字的纯说明段落每处 -4 |
| G2 扫描命中 | "不是X——是Y"句式每处 -2 |
| G4 扫描命中 | 反应场景模板化每处 -2 |
| G5 扫描超标 | 原著高频词 >1 个未变异 -2 |
| G9 扫描超标 | 盐值禁用词超标 -2/词 |
| G10 扫描超标 | 重复短语超标 -2/短语 |

### D2-B：短段落风格（权重 4 分）

| 检查项 | 扣分规则 |
|-------|---------|
| 短段落（≤3句）占比 < 50% | -2 |
| 句号密度 < 2.5（每千字） | -1 |
| 段落密度 < 7段/千字 | -1 |
| 单段 >150 字每处 | -2/处，最多 -3 |

---

### D3：爽点落地（权重 20 分）

执行流程：
1. 从章纲读取本章申报的爽点类型
2. 从总纲领 §六 读取该类型的专属触发公式
3. 按公式逐步检查正文

若章纲未标注爽点类型 → D3 得分 0。

| 检查项 | 扣分规则 |
|-------|---------|
| 本章是否包含章纲指定的主要爽点类型 | 缺失 -15 |
| 按该类型的专属公式逐步检查——每步是否存在 | 每缺失 1 步，扣 `floor(20 / 公式总步数)` 分 |
| 每步是否有变异（照搬 vs 合理变异） | 照搬公式字面每步 -1；合理变异每步 +1（单项最高 +3） |
| 是否有该类型对应的收束笔 | 缺失 -4 |
| 爽点类型与章纲标注不一致 | -3/处 |

---

### D4：人物设定一致性（权重 6 分）

| 检查项 | 扣分规则 |

| 配角行为是否符合其功能定位 | 每处不符合 -3 |
| 角色关系结构是否与总纲领一致 | 偏离 -4 |
| 主角是否处于正确的成长阶段 | 越阶行为 -5（归零） |

### D6：节奏结构（权重 9 分）

| 检查项 | 扣分 |
|-------|------|
| 开场超过 3 段还没进入正题 | -2 |

---

### D7：钩子质量（权重 10 分）

| 评分标准 | 分数 |
|---------|------|
| 优秀：悬念强，一句话引发强烈好奇 | 9~10 |
| 良好：有悬念，能驱动追读 | 6~8 |
| 及格：有收尾但悬念一般 | 3~4 |
| 不及格：平缓结束，无追读动力 | 0~2 |
| 无钩子 | 0 |

钩子检查清单：
- [ ] 是否在最后 1~3 段内完成？
- [ ] 是否属于 3 种模板之一（悬念钩/金句钩/反转钩）？
- [ ] 是否避免"欲知后事如何"等过时写法？

| 检查项 | 扣分规则 |

| 人名/地名/专有名词一致 | 不一致 -3/处 |

| 时间线连续 | 断裂 -3/处 |
| 现代术语泄露（G6扫描结果） | 命中 -3/处 |

### D9：原著相似度（权重 5 分）

| 检查项 | 扣分 |
|-------|------|
| 爽点序列雷同 | -3 |
| 主角对白句式雷同 | -3 |
| 总分 ≥4 | 标记"相似度警告" |

---

### D10：对话占比（权重 6 分）

统计方式：Grep 统计中文字数 + 引号内字数，`dialogue_ratio = 引号内字数 / 总汉字数`。

| 对话占比 | 得分公式 | 得分范围 |
|----------|---------|:---:|
| ≥25% | 6（满分） | 6 |
| [20%, 25%) | `3 + 3×(ratio-20)/5` | [3, 6) |
| [15%, 20%) | `0 + 3×(ratio-15)/5` | [0, 3) |
| <15% | 0 | 0 |

## 三、纪要压缩规则

纪要目标体积：≤15KB/章。

| 维度类型 | 得分条件 | 输出格式 | 体积预算 |
|---------|---------|---------|:---:|
| 满分维度 | 得分 = 满分 | 仅一行 `✅ D{X} {得分}/{满分}` | ~50B/条 |
| 轻微扣分 | 扣分 ≤1 分 | 两行：`⚠️ D{X} {得分}/{满分}` + `  - {扣分原因}` | ~150B/条 |
| 显著扣分 | 扣分 ≥2 分 | 展开：`🔴 D{X} {得分}/{满分}` + `  - {逐条扣分原因}` + `  - {修改建议}` | ~500B/条 |

禁止：满分维度不输出详细分析段落。仅 D3/D5/D6/D7/D10（五大权重维度，共 57 分）允许在显著扣分时展开分析。
