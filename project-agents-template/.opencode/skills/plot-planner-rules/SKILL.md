## ★ 零、确定性细化模式

当调用方传入了注入包内容时，进入「确定性细化模式」。注入包给你的是 WHAT（什么事件必须发生），你设计的是 HOW（如何发生——用什么场景、什么对话方向、什么节奏）。

### 0.1 卷规划时的行为变化

| 原有行为（标准模式） | 确定性细化模式 |
|-------------------|--------------|
| 从盐值 volume_rhythm_profile 设计卷级阶段划分 | 直接使用注入包的阶段划分 |
| 根据总纲领和进度推断本卷核心冲突 | 直接引用注入包§1声明的核心冲突 |
| 生成卷纲角色出场表 | 直接引用注入包§2的角色状态描述 |
| 规划本卷主线清单 | 从注入包事件链中提取主线里程碑 |

### 0.2 章纲规划时的行为变化

| 原有行为（标准模式） | 确定性细化模式 |
|-------------------|--------------|
| 推断本章核心事件 | 从注入包事件表读取"核心事件"，拆解为五段式 |
| 自行决定出场角色 | 从注入包事件表读取，可追加纯功能配角（不超过上限） |
| 自行决定爽点类型 | 从注入包读取"爽点类型"作为候选池，设计具体呈现 |
| 自行管理伏笔 | 从注入包读取伏笔ID，设计具体埋设/回收方式 |
| 自行管理跨章因果链 | 注入包已预先设计好因果链，只需确保因果过渡自然 |

**不变的部分**：五段式结构设计、章末钩子设计、标签覆盖分配、批次组织、§二~§九全部检查约束。

### 0.3 冲突处理

优先级规则：注入包 > 总纲领 > 盐值的模板默认值。若注入包与总纲领的「禁止改动底层逻辑清单」冲突 → 以总纲领为准，标注冲突。

### 0.4 模式触发条件

- 调用方传入注入包（至少含"本卷确定性事件链"章节）→ 确定性细化模式
- 调用方未传入 → 标准模式（降级）

## ★ L2 替换模式（DIFF_WARNING 章专用）

### 触发条件

1. 外部 DIFF_WARNING：注入包 §2.1 的 `diff_warning_chapters` 包含本章号
2. 内部同类型疲劳：本批次内连续 3 章以上主爽点类型相同 → 自动触发
3. 内部场景同质化：连续 2 章以上的场景类型相同 → 自动触发

### 替换边界：L2 vs L1

| ✅ 可替换（L2 范畴） | ❌ 不可替换（L1 范畴，仍归 destiny） |
|:---|:---|
| 事件发生的时空环境（早晨/深夜、室内/户外、晴/雨） | 核心事件本身 |
| 触发方式（主动触发/被动触发/偶然触发） | 事件的结果输出 |
| 配角反应层（增加原著没有的多层级反应细节） | 伏笔的埋设和回收 |
| 附加线索（在原事件骨架上叠加细节/暗记） | 出场角色 |
| 叙事角度（视角切换） | 注入包声明的"不可触碰内容" |

### L2 替换方向池

| 方向 | 说明 |
|:---|:---|
| 环境翻转 | 公开场景→私密场景，或反之 |
| 触发翻转 | 被动触发→主动触发，或反之 |
| 配角反应增强 | 增加多样化配角反应 |
| 附加线索植入 | 叠加原著没有的细线索 |
| 代价前置 | 爽点前展示代价 |
| 视角切换 | 旁观者视角开头再切回 |

### L2 替换后的安全性标注（每章章纲强制执行）

```markdown
## L2 替换记录
- 触发原因：□ DIFF_WARNING / □ 同类型爽点疲劳 / □ 场景同质化
- 使用方向：{方向名称}
- 替换范围：{具体描述}
- 未修改项确认：
  - [ ] 核心事件未变
  - [ ] 注入包伏笔 ID 未变
  - [ ] 注入包出场角色未变
  - [ ] 爽点类型未变
  - [ ] 注入包 §6 不可触碰内容零触碰
```

### 自查表

```markdown
### 场景骨架原创度自查

| 本章核心场景 | 场景类型 | 常见套路模式 | 本章差异化手段 | 差异化幅度 |
|---|---|---|---|---|
```

## 一、卷纲规划

### 1.1 赛道节奏模板选择

1. 读取 `project_salt.json`，检查 `volume_rhythm_profile` 字段
2. 若存在 → 使用 phases 作为本卷节奏阶段划分；读取 pacing_signature、milestone_types、forbidden_patterns
3. 若不存在（旧版盐值兼容）→ 降级使用兜底曲线，标注"⚠️ 使用兜底曲线"
4. 确定当前卷位，结合 volume_position_variants（若有）或通用卷位调节规则

### 1.2 卷纲结构模板

```
卷名：【第X卷 · {卷主题}】
覆盖章节：第N章 ~ 第N+M章

【卷级弧线】
● 起始状态：主角当前核心矛盾
● 本卷目标：阶段性成果
● 最终状态：本卷结束时的新位置

【赛道节奏签名】{pacing_signature}

【本卷里程碑类型】{选取 2~3 个}

【卷级爽点节奏曲线】（从 phases 逐条填入）
├─ {phase[0].name}（{range}）：{core_task}
├─ {phase[1].name}（{range}）：{core_task}
├─ ...
```

### 1.3 卷位动态调节

| 卷位 | 调节规则 |
|------|---------|
| 第一卷（开篇卷） | 首个 phase 压缩 ≤2 章；首个核心爽点最迟第 3 章前完成；节奏取赛道上限；每章结尾强制留钩子 |
| 中间卷 | 使用赛道标准曲线；卷首 2 章处理上卷余波+新冲突 |
| 末卷 | 取消所有"铺垫/积累/布局"类阶段；前 3 章开始收束暗线；最后一个 phase 扩展为连续高潮叠加 |

### 1.4 兜底默认曲线（旧版兼容）

仅当 project_salt.json 中不存在 volume_rhythm_profile 时使用：
```
├─ 开局引爆（1~3章）：快速入坑
├─ 布局期（4~12章）：暗线铺陈
├─ 中期高潮（13~16章）：中型反转/高潮
├─ 博弈期（17~24章）：压力累积
├─ 卷末大高潮（25~30章）：正面决战/格局重塑
```

## 二、批次规划（5 章节奏单元）

### 2.1 标准批次模板

```
┌──────────┬────────────┬──────────────────────────────────────────────┐
│ 章节      │ 角色定位    │ 爽点指数     │ 核心任务                      │
├──────────┼────────────┼──────────────────────────────────────────────┤
│ 第N章    │ 埋梗章      │ ★★          │ 铺垫日常+埋设伏笔+小悬念      │
│ 第N+1章  │ 酝酿章      │ ★★★        │ 冲突酝酿+主角暗中动作         │
│ 第N+2章  │ 小高潮章    │ ★★★★      │ 中型打脸/碾压/反转            │
│ 第N+3章  │ 过渡章      │ ★★          │ 事后续影响+人物关系推进       │
│ 第N+4章  │ 大高潮章    │ ★★★★★    │ 大型高潮/暗线收网/格局改变    │
└──────────┴────────────┴──────────────────────────────────────────────┘
```

### 2.2 批次变体

| 变体类型 | 适用场景 | 调整方式 |
|---------|---------|---------|
| 开局引爆型 | 新卷开头 3 章 | 压缩为 3 章：引爆→小高潮→后续 |
| 压力累积型 | 大高潮前 | 连续 2 章酝酿章，推迟小高潮至第 4 章 |
| 情感缓冲型 | 大高潮后 | 插入 1 章纯情感线/日常缓冲 |

### 2.3 批次间连续性检查（逐项勾选）

- [ ] 第 N 章的伏笔是否在第 N+4 章前得到回应或推进？
- [ ] 角色状态是否在 5 章内有可感知的变化？
- [ ] 核心冲突是否有推进（不能 5 章下来原地踏步）？
- [ ] 标签覆盖是否在批次内轮换？
- [ ] **爽点类型是否轮换**：本批次内任意连续 3 章不得为同一爽点类型；前 3 章主爽点类型须 ≥2 种不同类型
- [ ] **钩子类型是否轮换**：连续 3 章不得同为悬念钩，须强制插入金句钩/反转钩/情绪钩

### 钩子类型分配表（批次级·必填）

| 钩子类型 | 本批次分配数量 | 占比 | 硬约束 |
|---------|-------------|------|--------|
| 悬念钩 | __个 | ≤50% | 不得超过总章数的 50% |
| 金句钩 | __个 | ≥20% | 不得少于总章数的 20%，全书≥3个 |
| 反转钩 | __个 | ≥10% | 至少 1 个 |
| 情绪钩/其他 | __个 | 余量 | — |

### 更多批次检查

- [ ] 钩子载体是否多样化：若 5 章内 ≥3 章依赖同质载体，须替换
- [ ] 爽点空窗检查：连续 2 章无任何爽点 → 插入至少 1 个中型即时爽点
- [ ] 配角弧线规划：出场 ≥3 场的命名配角须标注 independent_motivation
- [ ] 金手指呈现是否变体：相邻两次须取不同的 presentation_variant
- [ ] 叙事模式变体检查：≥3 章高潮为同类结构 → 设计叙事变体
- [ ] 场景骨架原创度批次检查：≥2 章差异幅度为"低"→ 判定场景同质化风险
- [ ] 同类场景叙事曲线轮换：≥2 章同类场景 → 从 4 种叙事变体中选 ≥2 种轮换
- [ ] 反派转变弧线前置：转变前 ≥2 章埋设暗示性伏笔
- [ ] 配角复杂面落地检查：盐值标注了 independent_motivation 但章纲无对应场景 → 须增加
- [ ] 信息获取链是否自洽：主角报出的信息必须在已知信息清单中有来源
- [ ] 五段式占比是否合规：开场≤10%、收尾≤15%、高潮≥15%

## 三、角色状态管理

### 确定性细化模式 → 角色状态校验模式
1. 从注入包 §2 读取每个出场角色的本卷初/末状态
2. 对每个出场角色执行一致性校验：本章事件是否自洽？变化方向是否朝末状态收敛？
3. 不通过则标注冲突

### 标准模式 → 角色成长阶段追踪

| 成长阶段 | 章节范围参考 | 阶段特征 | 不允许出现的写作行为 |
|---------|------------|---------|-------------------|
| ① 隐忍蛰伏 | 1~30 章 | 暗中观察，不暴露实力 | 不可在家族内部公开身份 |
| ② 暗中布局 | 30~80 章 | 代理网络建立 | 不可正面对抗 |
| ③ 小范围展露 | 80~150 章 | 对部分盟友暴露 | 不可全面公开 |
| ④ 代理人战争 | 150~250 章 | 通过代理人影响 | 本人不可直接出面 |
| ⑤ 半公开博弈 | 250~350 章 | 正式场合亮牌 | 不可暴露终极底牌 |
| ⑥ 全面掌控 | 350 章+ | 终极对决 | — |

## 四、伏笔管理

### 确定性细化模式 → 伏笔引用模式

使用 destiny 的伏笔体系（禁止生成 F-xxx 格式 ID）。每章章纲必填伏笔引用登记表：

```
┌──────────┬────────┬──────────┬──────────┬────────────────────┐
│ 伏笔ID   │ 级别   │ 内容摘要  │ 本章操作  │ 本章埋设/回收方式    │
│(destiny) │(A/B/C) │          │(埋设/回收)│                     │
└──────────┴────────┴──────────┴──────────┴────────────────────┘
```

### 标准模式 → 伏笔管理系统

| 类型 | 定义 | 最长保留期限 |
|------|------|-------------|
| 悬念伏 | 引发读者好奇的悬念 | 3~8 章内回收 |
| 道具伏 | 物品/信息在后续发挥作用 | 10~20 章 |
| 成长伏 | 主角学习/积累能力的痕迹 | 一卷内回收 |
| 背景伏 | 角色身世/历史的暗示 | 多卷后回收 |

管理规则：
- 每章章纲末尾列出新埋设和已回收的伏笔
- 同一批次内不允许超过 3 个未回收的悬念伏
- 伏笔回收必须有"读者恍然大悟"的效果

## 五、爽点预算机制

### 确定性细化模式 → 事件链反算模式

从注入包 §3 事件链的"爽点类型"列统计，检查是否符合盐值 pleasure_rotation 约束。单章预算从事件链读取"爽点类型"作为主推。

### 标准模式 → 标准预算模板

```
本章爽点预算：
□ 主要爽点（必选）：
   - 类型：________
   - 位置：________段
   - 强度：________（轻量/中型/重量级）
   - ⚠️ 轮换校验：主爽点类型不得与前两章相同
□ 次要爽点（可选）
□ 辅助爽点（可选）
□ 金手指呈现变体：选用 variant________（须与本批次相邻一次不同）
```

| 章节角色 | 主要爽点强度 | 次要爽点 |
|---------|------------|---------|
| 埋梗章（★★） | 轻量或无需 | 可无 |
| 酝酿章（★★★） | 中型 | 1 个轻量 |
| 小高潮章（★★★★） | 重量级 | 1~2 个中型 |
| 过渡章（★★） | 轻量 | 可无 |
| 大高潮章（★★★★★） | 重量级 2 个 | 2~3 个中型 |

## 六、章节间因果约束

每章章纲末尾必须标注"下章触发条件"：
```
本章事件 → 导致后果 → 下章触发条件
```

违规红线：
- ❌ 章纲事件结束后，下一章从完全无关的场景开始
- ❌ 同一冲突反复使用超过 3 次而不升级
- ❌ 角色状态在连续多章内无任何变化

## 七、章内信息流约束（防降智·防巧合）

### 7.1 信息获取清单（每章必填）

```
## 角色信息获取清单
- 主角已知信息（截至本章末）：
  - [信息A]：来源=第X章/场景Y（方式：亲眼见/亲耳听/推算/他人告知）
- 本章新获取信息：
  - [信息C]：获取方式=______（必须有对应场景）
- 本章主角向他人报出的信息：
  - [信息D]：向______报出 → 前提：主角已在第X章获取
  - ⚠️ 若报出的信息在"已知信息"中找不到来源 → 须补充或删除
```

### 7.2 巧合回避检查

| 场景类型 | ❌ 禁止设计 | ✅ 合理设计 |
|---------|-----------|-----------|
| 窃听/偷听机密 | 反派在公共场合大声讨论核心阴谋 | 反派在私密据点讨论，主角基于前世记忆定位后潜入 |
| 偶遇关键人物 | 主角随机走动就撞见关键NPC | 主角基于前世记忆知道NPC行动规律，主动前往 |
| 意外发现线索 | 翻找物品时恰好翻到关键证据 | 主角基于前世记忆锁定搜索范围 |

### 7.3 配角智商下限保护

- 禁止设计"反派/配角在无逻辑的情况下暴露关键信息"的场景
- 若配角需通过对话暴露信息，必须注明暴露动机（炫耀/试探/酒后失言/被激将）
- 公共场合禁止讨论核心阴谋

## 八、五段式占比硬约束

| 段落 | 占比上限 | 超标后果 |
|------|---------|---------|
| ① 开场 | ≤10%（闪回类≤8%） | 判定开场冗长 |
| ② 铺垫 | ≤50% | 判定节奏拖沓 |
| ⑤ 收尾 | ≤15% | 判定收尾过长 |

收尾段内容限制：仅允许放置"事件余波（≤2段）+ 下章钩子（1-3段）"。高潮段（④）占比下限 = 15%。
