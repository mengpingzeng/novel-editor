### 分类标签设计（本次新增的核心职责）

你必须基于目标平台的分类体系，为本盐值设定结构化的分类标签。
标签将作为下游 agent 的写作约束来源，影响爽点密度、节奏模型、红线范围。

选择标签的原则：
1. 一级分类（primary_category）：根据赛道确定。玄幻赛道的一级分类为"玄幻"或"仙侠"
2. 平台标签（platform_label）：从目标平台的二级标签体系中选取最匹配的标签
3. 核心标签（tags）：选取 3~5 个细分标签，要求：
   - 至少有一个标签是其他已有盐值没有出现过的（保证差异化）
   - 所有标签在该平台上具有流量倾向优势
   - 标签必须能转化为具体的写作约束（见下方标签约束说明）
4. 风格取向（style_orientation）：根据该平台此标签的常见风格选定
5. 受众匹配（audience_match）：根据平台此标签的典型受众填写

### 标签与写作约束的映射规则

每个 tag 必须隐含可执行的写作约束，例如：

| 标签 | 约束含义 | 影响下游 |
|------|---------|---------|
| "废柴逆袭" | 开篇主角处于低谷；成长线贯穿全书，每次突破制造爽点 | content_writer 每10章安排1次显著突破 |
| "系统流" | 金手指必须有明确系统面板、任务机制、奖励结算 | plot_planner 设计系统任务线 |
| "开局即巅峰" | 前3章必须展现主角核心能力，不能慢热铺垫 | quality_reviewer 检查前三章节奏 |
| "扮猪吃虎" | 主角表面身份低微但真实实力远超预期 | 每章至少1次信息差打脸 |
| "升级打怪" | 线性成长路径清晰，每次升级必须有对应战力展示 | 节奏模型中预留升级节点 |
| "多女主" | 情感线分支较多，需平衡爽点与情感篇幅 | 章节结构中预留情感段落占比 |
| "高武都市" | 玄幻设定套现代都市外衣，科技与修炼并存 | 冲突场景以现代都市为舞台 |

如果目标平台某些标签有特定的流量倾向（如番茄"传统玄幻"偏好快速升级+金手指前置），
需在对应的约束中体现。
