### 分类标签设计

你必须基于目标平台的分类体系，为本盐值设定结构化的分类标签。
标签将作为下游 agent 的写作约束来源，影响爽点密度、节奏模型、红线范围。

选择标签的原则：
1. 一级分类（primary_category）：历史赛道的一级分类为"历史"
2. 平台标签（platform_label）：从目标平台的二级标签体系中选取最匹配的标签
3. 核心标签（tags）：选取 3~5 个细分标签，要求：
   - 至少有一个标签是其他已有盐值没有出现过的（保证差异化）
   - 所有标签在该平台上具有流量倾向优势
   - 标签必须能转化为具体的写作约束（见下方标签约束说明）
4. 风格取向（style_orientation）：根据该平台此标签的常见风格选定
5. 受众匹配（audience_match）：根据平台此标签的典型受众填写

### 标签与写作约束的映射规则

每个 tag 必须隐含可执行的写作约束，例如：

| 标签 | 约束含义 | 影响下游 |
|------|---------|---------|
| "历史穿越" | 现代人穿越到古代，利用现代知识降维打击为爽点核心 | content_writer 每章至少1次现代知识应用场景 |
| "争霸" | 以逐鹿天下为主线，战争策略、外交纵横、治国安邦为主要内容 | plot_planner 设计阶段性征服目标 |
| "种田" | 侧重经济建设、科技发展、民生改善，节奏可略慢于争霸流 | 爽点密度以"发展成果展示"为主 |
| "权谋" | 朝堂博弈、权术斗争、派系制衡为核心冲突，智力碾压为爽点 | quality_reviewer 检查权谋逻辑是否自洽 |
| "架空历史" | 虚构朝代设定，不受真实历史约束但需内部逻辑自洽 | world_mapping 中需完整构建朝代背景 |
| "军事战争" | 以战争场景为主，兵法运用、以少胜多为高潮爽点 | 每卷至少1场大型战争/战役高潮 |
| "科技兴国" | 主角引入现代科技推动时代变革，工业/农业/军事全面升级 | 章节节奏中安排阶段性科技突破节点 |

历史赛道红线提醒（写入约束）：
- 真实历史人物不可过度丑化或魔改（七猫审核严格）
- 涉政内容需谨慎（特别是近现代史）
- 尊重历史主流价值观，避免明显的历史虚无主义
