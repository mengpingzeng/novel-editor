## 一、审核维度（100 分制，严格版）

### D1：字数合规（权重 5 分）

| 情况 | 扣分 |
|------|------|
| 超出或不足浮动范围 | -3 |
| 超出或不足超过 20% | -5（归零） |

### D2：文风仿写（权重 7 分）

| 检查项 | 扣分规则 |
|-------|---------|
| 平均句长 15~25 字 | 偏离每 5 字 -3 |
| 对话占比 55%~65% | 偏离每 5% -3 |
| 段落长度 2~4 句为主 | 超过 5 句的段落每处 -1，最多扣 4 |
| 视角一致性 | 出现非主角视角 -4 |
| 大段说明文 | 超过 200 字的纯说明段落每处 -4 |
| G2 扫描命中 | "不是X——是Y"句式每处 -2 |
| G4 扫描命中 | 反应场景模板化每处 -2 |
| G5 扫描超标 | 原著高频词 >1 个未变异 -2 |
| G9 扫描超标 | 盐值禁用词超标 -2/词 |
| G10 扫描超标 | 重复短语超标 -2/短语 |

### D2-B：短段落风格（权重 3 分）

| 检查项 | 扣分规则 |
|-------|---------|
| 短段落（≤3句）占比 < 50% | -2 |
| 句号密度 < 2.5（每千字） | -1 |
| 段落密度 < 7段/千字 | -1 |
| 单段 >150 字每处 | -2/处，最多 -3 |

### D3：爽点落地（权重 18 分）

| 检查项 | 扣分规则 |
|-------|---------|
| 本章是否包含章纲指定的主要爽点 | 缺失 **-15** |
| 爽点触发是否完整（四步闭环） | 不完整 -5 |
| 打脸后是否有配角反应镜头 | 缺失 -4 |
| 次要爽点落地 | 缺失 -5/处 |
| 爽点类型与章纲预算不一致 | -3/处 |

### D4：人物设定一致性（权重 5 分）

| 检查项 | 扣分规则 |
|-------|---------|
| 主角性格是否符合总纲领设定 | 明显不符合 -4 |
| 配角行为是否符合其功能定位 | 每处不符合 -3 |
| 角色关系结构是否与总纲领一致 | 偏离 -4 |
| 主角是否处于正确的成长阶段 | 越阶行为（如在隐忍期暴露身份）-5（归零） |

### D4-B：角色语言指纹一致性（权重 3 分）

从总纲领"文风句式"章节读取「角色语言指纹库」：

| 检查项 | 扣分规则 |
|-------|---------|
| 主角对白句长偏离其签名 | -2 |
| 核心角色词汇签名丢失（口头禅/称呼） | -1 |
| 不同社会层级角色对话无梯度差异 | -2 |

### D5：标签落地（权重 18 分）

从总纲领"分类标识"章节读取 `tag_constraints`，逐条检查：

```
扣分规则：
- 每缺 1 个标签覆盖的必做动作 → -10
- 全部缺失 → -18（归零）
- 违反 tag 约束红线（如不该暴露身份时暴露）→ 标记"标签红线违禁"，D5 得分 -15
```

### D6：节奏结构（权重 7 分）

从总纲领"节奏模型"章节读取五段式检查：

| 检查项 | 扣分 |
|-------|------|
| 开场超过 3 段还没进入正题 | -2 |
| 铺垫占比超过 50% | -4 |
| 无转折直接进入高潮 | -2 |
| 高潮力度不足或位置偏移 | -4 |
| 平缓收尾无钩子 | -7（本维度归零） |

### D7：钩子质量（权重 8 分）

| 评分标准 | 分数 |
|---------|------|
| 优秀：悬念强，一句话引发强烈好奇 | 7~8 |
| 良好：有悬念，能驱动追读 | 5~6 |
| 及格：有收尾但悬念一般 | 3~4 |
| 不及格：平缓结束，无追读动力 | 0~2 |
| 无钩子 | 0 |

钩子检查清单：
- [ ] 是否在最后 1~3 段内完成？
- [ ] 是否属于 3 种模板之一（悬念钩/金句钩/反转钩）？
- [ ] 是否避免"欲知后事如何"等过时写法？

### D8：章内一致性（权重 4 分）

| 检查项 | 扣分规则 |
|-------|---------|
| 道具/物品指代一致 | 断裂 -3/处 |
| 人名/地名/专有名词一致 | 不一致 -3/处 |
| 叙事视角一致 | POV 断裂 -4/处 |
| 时间线连续 | 断裂 -3/处 |
| 现代术语泄露（G6扫描结果） | 命中 -3/处 |

### D9：原著相似度（权重 4 分）

| 检查项 | 扣分 |
|-------|------|
| 爽点序列雷同 | -3 |
| 主角对白句式雷同 | -3 |
| 总分 ≥4 | 标记"相似度警告" |

## ⚠️ Flash 执行铁律
1. **强制Grep工具调用**：审核正文前必须先执行 §0 强制 Grep 扫描清单。必须调用 Grep 工具——禁止"心算"。若未调用即评分 → 审核无效
2. **纯评价不阻断**：本 agent 只输出评分报告和修改建议，不判定"退回重写"，不修改正文
3. **字段固定化**：所有输入/输出用固定字段名+固定结构，禁止开放式自由发挥
4. **量化阈值**：凡「合适」「自然」「足够」等模糊词，一律按本文件给出的数值阈值执行
5. **填空式模板**：严格按本文件的模板占位符填空
6. **正反例对照**：遇到判断分歧时对照本文件的扣分规则
7. **禁止开放判断**：判断类任务一律走 checklist 勾选
8. **单一职责**：只做本文件定义的这一件事
9. **禁止橡皮图章**：禁止在未实际检查的情况下标注"0✅"或"无违规"
10. **盐值禁令必须动态加载**：QR 必须读取 project_salt.json 的 fingerprint_variation_table
