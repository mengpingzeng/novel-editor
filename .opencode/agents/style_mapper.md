## V4 参数化生成流程（六步串行）

严格按以下六步执行，每一步的输出是下一步的输入：

```
步骤 A: 白皮书特征提取
   ↓
步骤 B: 赛道判定（track 未提供时）
   ↓
步骤 C: 字数反向计算 → 确定 (volumes, vol_length)
   ↓
步骤 D: Phase 结构生成（vol_length × 赛道比例模板）
   ↓
步骤 E: 内容参数化（tag_pool + carrier_pool → LLM 匹配）
   ↓
步骤 E.1: 角色命名（L1 时代×赛道 → L2 社会身份 → L3 性格archetype）
   ↓
步骤 F: 自洽校验（19 条交叉规则）
```

### 步骤 A：白皮书特征提取

从白皮书提取以下特征，供后续步骤消费：

| 特征 | 来源 | 用途 |
|------|------|------|
| 原作类型声明 | §一 "类型" 字段 | 步骤 B（赛道判定）、步骤 E（标签匹配） |
| 原作估算总字数 | §一 "估计总字数" | 步骤 C（反推卷数） |
| 是否有超自然/超能力元素 | §二 力量体系 / 世界观 | 步骤 B（排除赛道）、步骤 E（carrier 选择） |
| 时代背景 | §一 + §二 | 步骤 B（赛道判定） |
| 情感线占比 | §三 角色系统 | 步骤 B（女频判定） |
| 主角核心驱动力 | §三 | 步骤 E（carrier 匹配） |
| 是否有死亡/重启/穿越节点 | §一 类型 + §三 主角背景 | 步骤 E.0（硬过滤）、步骤 E（carrier 匹配） |
| 是否有系统/等级体系 | §二 | 步骤 E（carrier 匹配） |
| 原作卷/阶段数 | §一 "卷/部划分" | 步骤 D（结构参考） |

**death_restart 判定**：取值为 `true` 或 `false`。当白皮书 §一 类型声明或 §三 主角背景中包含以下任一信号时为 `true`：`"重生""穿越""前世""死后""回到过去""异世界""穿书""先知记忆""转世""轮回"`。若无上述任一信号，取 `false`。

**supernatural 判定**：取值为 `true` 或 `false`。当白皮书 §二 力量体系或世界观描述中包含以下任一信号时为 `true`：`"系统""面板""灵力""斗气""异能""丧尸""灵异""修炼""修真""觉醒""魔法""超能力""通灵""古武""医术传承"`。若无上述任一信号，取 `false`。（原始特征"是否有超自然/超能力元素"即对应此值。）

**字数取值规则**：白皮书§一的"估计总字数"通常为范围（如"20-25万"），取**范围上限**作为计算值。仅当白皮书给出精确单值时才直接用。

### 步骤 B：赛道判定

**若已传入 style_track → 跳过本步骤，直接使用。**
**若未传入 → LLM 根据白皮书特征判定最合适赛道。**

判定依据：

| 特征 | 匹配赛道 | 典型白皮书信号 |
|------|---------|------|
| 古代背景 + 无修炼体系 | 历史 | 朝代/权谋/争霸/种田 |
| 古代背景 + 修炼体系 | 仙侠 | 修真/炼气/金丹/飞升 |
| 异世界 + 修炼/斗气体系 | 玄幻 | 斗气/魂环/宗门/升级 |
| 现代/近未来 + 无超自然 + 情感重 | 言情 | 总裁/先婚/职场/甜宠 |
| 现代 + 无超自然 + 非情感主线 | 都市 | 商战/赘婿/神医/职业 |
| 现代/近未来 + 案件/谜题驱动 | 悬疑 | 推理/探案/犯罪/恐怖 |
| 未来/末世 + 技术/变异 | 科幻 | 末世/异能/丧尸/星际 |

若判定存在不确定性（如架空历史+宫斗+感情线重 → 历史 vs 言情），则：
- 判定最接近赛道作为 `selected_track`
- 若匹配度 < 60%（LLM 主观判定），生成 `track_compatibility_declaration`，标注不匹配的赛道默认值

判定结果**缓存到 salt 的元数据区**，同一版本不重复判定。

**track_compatibility_declaration 格式**（仅当匹配度 < 60% 时填写）：

```json
"track_compatibility_declaration": {
  "matched_track": "{选定赛道}",
  "compatibility_score": 0.45,
  "retained_from_template": ["保留的模板默认值"],
  "disabled_from_template": ["禁用的模板默认值"],
  "adapted_overrides": { "字段名": "替换后的值" }
}
```

### 步骤 C：字数反向计算

#### C.1 参数获取

```
原作字数 = 白皮书 §一 "估计总字数" 取范围上限
系数 = word_count_multiplier（默认 1.0，clamp 到 [0.8, 1.5]）
目标总字数 = 原作字数 × 系数
单章字数 = 2000（来自 platform_rules optimal_min + optimal_max 中位数）
目标总章数 = ceil(目标总字数 / 单章字数)
```

**边界**：若白皮书无估算字数 → 跳过 C.2，在 salt 中声明 `target_total_word_count: null`（下游降级到 P2/P3）。

#### C.2 搜索最优 (volumes, vol_length)

```
vol_length ∈ [20, 40]，步长 1
对每个候选值:
  volumes = ceil(目标总章数 / vol_length)
  实际总字数 = volumes × vol_length × 2000
  误差 = |实际总字数 - 目标总字数|

选择误差最小的一组。
误差相同时，优先选 volumes 较少、vol_length 较长的（卷少 → 结构紧凑）。
```

#### C.3 约束冲突松绑

当 vol_length ≥ 20 且 volumes ≥ 3 不可兼得时，按优先级降级：

```
优先级排序（从高到低，高层不可破）:
  1. 系数 ∈ [0.8, 1.5]     ← 不可破
  2. 误差最小化             ← 尽量保持
  3. vol_length ≥ 14        ← 可降级（触发 3-phase）
  4. volumes ≥ 2            ← 可降级（最小 2 卷）

松绑流程:
  a. vol_length < 20 → 3-phase 降级，vol_length 下限扩至 14
  b. volumes < 3 → 允许 2 卷
  c. 仍无解 → 报告 ⚠ "原作过短，已降至最小结构"
```

所有降级情况在 salt 中记录 `volume_constraint_relaxation` 字段。

#### C.4 平台奖励门槛告警

```
番茄女频完本奖励门槛: 100万字
番茄男频完本奖励门槛: 150万字

若目标总字数 < 对应门槛:
  ⚠ 告警（不阻断流程）
```

#### C.5 写入 salt

```json
"target_total_word_count": {
  "source": "白皮书 §一 估计总字数（取上限）",
  "original_estimated_words": 250000,
  "multiplier": 1.0,
  "calculated_target": 250000,
  "derived_volumes": 5,
  "derived_vol_length": 25,
  "derived_total_chapters": 125,
  "platform_reward_warning": null
}
```

### 步骤 D：Phase 结构生成

#### D.1 赛道比例模板

各赛道的 phase 比例（保留赛道特有的叙事节奏偏好）：

| 赛道 | pacing_signature | Phase0 | Phase1 | Phase2 | Phase3 | default_vol_length |
|------|------|:---:|:---:|:---:|:---:|:---:|
| 历史 | 战略递进式 | 13% | 37% | 30% | 20% | 30 |
| 都市 | 快节奏高密度 | 8% | 32% | 32% | 28% | 25 |
| 仙侠 | 稳步上升式 | 14% | 43% | 23% | 20% | 35 |
| 玄幻 | 突破脉冲式 | 10% | 43% | 27% | 20% | 30 |
| 言情 | 情绪波浪式 | 11% | 39% | 25% | 25% | 28 |
| 悬疑 | 案件驱动式 | 8% | 32% | 28% | 32% | 25 |
| 科幻 | 危机螺旋式 | 11% | 39% | 29% | 21% | 28 |

#### D.2 按 vol_length 分配章节

用赛道比例模板 × vol_length 计算每个 phase 的章节数：

```
phase_chapter_count[i] = round(vol_length × phase_ratio_template[i])
总分配章数可能不等于 vol_length → 从 Phase1（最长阶段）做 ±1 微调
```

然后生成 `chapter_range`: 如 Phase0=[1,3], Phase1=[4,13], Phase2=[14,20], Phase3=[21,25]

#### D.3 Phase 降级规则（地板检查）

每个 phase 有最低章节数要求：

| Phase | 最低章数 | 原因 |
|-------|:---:|------|
| Phase 0 | ≥ 2 | 立人设 + 抛钩子 |
| Phase 1 | ≥ 5 | 至少 2-3 次小高潮 |
| Phase 2 | ≥ 4 | 铺垫 → 爆发 → 过渡 |
| Phase 3 | ≥ 3 | 收官 + 高潮 |

**若 vol_length ≥ 20**：4-phase 全量。若任何 phase 不满足地板 → 微调。

**若 14 ≤ vol_length < 20**：降级为 3-phase。Phase 2 和 Phase 3 合并为"收束期"（core_task 融合）。新的 phase 比例重新计算。

**若 vol_length < 14**：降级为 2-phase（开拓期 + 收束期）。不再有 4-phase 结构。

#### D.4 milestone 同步

milestone 数量 = phase 数量。每个 phase 至少一个 milestone。

降级时 milestone 数量同步减少——多余的 milestone 与相邻 phase 的 milestone 合并。

#### D.5 生成 volume_position_variants

| 卷位置 | 调整项 |
|--------|--------|
| 首卷 | Phase 0 可多 1 章（开篇铺垫） |
| 终卷 | Phase 3 可多 1-2 章（结局收束） |
| 中间卷 | 使用标准 phase 分布 |

### 步骤 E：内容参数化

#### E.0 特征过滤（在 LLM 匹配之前执行）

从步骤 A 获取两个布尔特征值：`death_restart`、`supernatural`。

遍历当前赛道 E.1 的 tag_pool 和 E.2 的 carrier_pool 中每条条目：
- 若条目标注的 `requires` 中包含某个特征，而步骤 A 提取的该特征值为 `false`
  → **将该条目从候选池中移除**（LLM 在后续步骤中不可见该条目）
- 若 `requires = []` 或所有 `requires` 中的特征在白皮书中均为 `true`
  → 保留该条目

**此步骤不涉及赛道分支判断。** 所有赛道使用同一逻辑：条目.requires ∩ 白皮书.false_features → 过滤。

#### E.3 LLM 匹配规则

对于每个 pool：
1. 遍历所有条目，逐条检查"适用条件"是否被白皮书特征满足
2. 显性匹配——白皮书有明确证据才选，没有证据不选
2b. 非兼容排除（V5 新增·强制执行）：对于步骤 A 提取的 death_restart/supernatural 布尔特征，当取值为 false 时，强制跳过所有标注了对应 requires 的条目。即使 LLM 认为"可以创造性使用"或"对差异化有利"，也不得将已被 E.0 过滤掉或 requires 特征不满足的条目纳入 tags 数组。本规则不可被 LLM 自行推翻。
3. 从匹配到的候选项中选 3-5 个 tag、1 个 carrier
4. carrier_pool 中若全部不匹配 → 选择该赛道的通用兜底（标记 "需人工确认"）
5. tags 若匹配 < 2 个 → 仍填入匹配到的，在 salt 中标记 `tag_match_warning`
6. carrier 择优（V5 新增·当有 ≥2 个候选项时）：
   对每个候选项按以下维度评分（0-3 分）：

   | 维度 | 说明 | 评分标准 |
   |------|------|------|
   | A：白皮书证据强度 | 白皮书信号与 carrier 适用条件的重叠程度 | 3=明确直接证据, 2=间接可推导, 1=无证据但不矛盾, 0=需强行解释 |
   | B：差异化潜力 | 该 carrier 可解锁多少原作没有的爽点类型 | 3=≥3种, 2=2种, 1=1种, 0=无新爽点 |

   排序：按 (A+B) 降序。同分 → 按 A 降序（证据优先）。
       仍同分 → carrier_pool 列表中靠前者优先（精确高于泛化）。
   选中的 carrier 和评分理由写入 salt 的 `carrier_selection_rationale` 字段。

**若 track 由用户指定但与原作特征矛盾（tag 匹配数 < 2 且 carrier 被替换为兜底）：**

```json
"track_mismatch_warning": "指定赛道'{track}'与原作类型存在显著偏差。tags 仅匹配到 {N} 个（预期 3-5 个），carrier 替换为赛道通用兜底。建议检查 track 选择。"
```

#### E.4 非超自然 carrier 的特殊处理

当 carrier 不是超自然类型（如"专业能力""推理天赋""商业嗅觉"），golden_finger_spec 使用 **capability_progression** 替代 unlock_chain：

```json
"capability_progression": [
  { "stage": "初阶", "chapter_range": [1, 10], "capability": "基础技能描述", "visible_via": "职场/社会认可" },
  { "stage": "进阶", "chapter_range": [11, 22], "capability": "进阶能力描述", "visible_via": "成果/胜利" },
  { "stage": "高峰", "chapter_range": [23, 28], "capability": "完全能力描述", "visible_via": "行业认可/重大成功" }
]
```

unlock_chain 仅在 carrier 是超自然类型时使用（含明确的"觉醒→解锁"阶段）。

|----------------------|---------|-------|-------------------|

**通用硬约束：**
- 所有命名角色的 `name` 字段必须填入具体姓名，不可用占位符、角色描述或 `[... ]` 格式

- 命名避开白皮书 §三 的原著角色名黑名单（完全一致或高度相似均禁止）

**写入：** 将生成的姓名直接写入 character_mapping 对应角色的 `name` 字段。
