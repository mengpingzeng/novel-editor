# quality_reviewer 字数限制严格化方案

> 编写日期: 2026-07-04
> 目标版本: v2
> 涉及 Agent: `quality_reviewer`（主改），`book_factory`（行为影响分析）

---

## 一、问题分析

### 1.1 当前 D1 评分逻辑（过松）

```markdown
### D1：字数合规（权重 5 分）

| 情况 | 扣分 |
|------|------|
| 超出或不足浮动范围 | -3 |
| 超出或不足超过 20% | -5（归零） |
```

**核心缺陷**：D1 最多扣 5 分（权重仅 5），即使字数完全失控，剩余 9 个维度仍可凑出 60 分，导致评分形同虚设。

示例场景（番茄平台，optimal 1800-2200，浮动范围 ±200）：
- 实际 1000 字 → D1 = 0（超 20%），但 D3+D5 两个维度做好就拿 36 分，再加其他维度轻松过 60
- 实际 3500 字 → 同上

### 1.2 根本原因

- D1 惩罚上限过低（仅 5 分权重），无"一票否决"机制
- 扣分是阶梯式（-3 / -5），缺乏连续性过渡
- 对极端偏离子数没有整章作废级别的惩罚

## 二、新方案设计

### 2.1 核心原则

**字数严重偏离 → 整章作废（总分 = 0），而非仅在 D1 维度扣分。**

引入三道防线：

| 防线 | 触发条件 | 惩罚级别 |
|------|---------|---------|
| 硬性不合格 | `< 1000` 或 `> 3000` | **整篇 0 分** |
| 过渡区间 | `[1000, 1500)` 或 `(2500, 3000]` | **D1 线性扣分（0 ~ 5）** |
| 满分区间 | `[1500, 2500]` | **D1 = 5（满分）** |

### 2.2 判定流程图

```
实际字数
  │
  ├─ < 1000 ──────────────────→ 整篇 0 分（硬性不合格）
  │
  ├─ [1000, 1500) ──→ D1 = 5 × (字数 - 1000) / 500
  │
  ├─ [1500, 2500] ───→ D1 = 5（满分）
  │
  ├─ (2500, 3000] ──→ D1 = 5 × (3000 - 字数) / 500
  │
  └─ > 3000 ──────────────────→ 整篇 0 分（硬性不合格）
```

### 2.3 计算公式（精确）

```
IF 字数 < 1000:
    整篇总分 = 0，所有维度全部归零
    标记理由："字数不达标-直接不合格（实际{字数}字，低于下限1000字）"

ELIF 1000 ≤ 字数 < 1500:
    D1 = round(5 × (字数 - 1000) / 500, 1)
    其他 9 个维度正常评分

ELIF 1500 ≤ 字数 ≤ 2500:
    D1 = 5
    其他 9 个维度正常评分

ELIF 2500 < 字数 ≤ 3000:
    D1 = round(5 × (3000 - 字数) / 500, 1)
    其他 9 个维度正常评分

ELIF 字数 > 3000:
    整篇总分 = 0，所有维度全部归零
    标记理由："字数超标-直接不合格（实际{字数}字，上限3000字=2500×120%）"
```

### 2.4 D1 分值曲线可视化

```
D1
 5 ┤                          ╭──────────╮
   │                         ╱            ╲
 4 ┤                        ╱              ╲
   │                       ╱                ╲
 3 ┤                      ╱                  ╲
   │                     ╱                    ╲
 2 ┤                    ╱                      ╲
   │                   ╱                        ╲
 1 ┤                  ╱                          ╲
   │                 ╱                            ╲
 0 ┤────────────────╱                              ╲──────────────
   0   500  1000   1500   2000   2500   3000   3500   4000
        ↑              ↑              ↑       ↑
    (整篇0分)      满分区起点      满分区终点  (整篇0分)
```

### 2.5 阈值说明

| 阈值 | 含义 | 来源 |
|------|------|------|
| 1000 | 绝对下限 | 容错边界（低于此值可判定为生成失败） |
| 1500 | 满分下限 | 网文单章最低有效阅读量的经验值 |
| 2500 | 满分子限 | 网文单章最佳阅读量的经验值 |
| 3000 | 绝对上限 | `2500 × 1.2`，严格意义上超过 20% 即判定失控 |
| 500 | 过渡区间宽度 | `1500 - 1000 = 500`，`3000 - 2500 = 500`，线性过渡等距 |

**注意**：新阈值是固定绝对值，不再从《仿写衍生总纲领.md》的"平台适配"章节读取 `optimal_min`/`optimal_max`/浮动范围来计算。原平台适配信息仍保留在总纲领中，但仅作参考，不再参与 D1 评分计算。

#### 3.2.3 第 94-99 行 — D1 评分表（完全重写）

**当前文本**：
```markdown

| 情况 | 扣分 |
|------|------|
| 超出或不足浮动范围 | -3 |
| 超出或不足超过 20% | -5（归零） |
```

**替换为**：
```markdown
### D1：字数合规（权重 5 分）

字数阈值：

| 阈值 | 值 |
|------|-----|
| 绝对下限（低于 → 整篇 0 分） | 1000 字 |
| 满分下限 | 1500 字 |
| 满分上限 | 2500 字 |
| 绝对上限（超过 → 整篇 0 分） | 3000 字（= 2500 × 120%） |

D1 得分规则：

| 实际字数 | D1 得分 | 说明 |
|----------|---------|------|
| < 1000 | 整篇 0 分 | 硬性不合格，所有维度归零 |
| [1000, 1500) | `5 × (字数 - 1000) / 500` | 线性过渡，保留 1 位小数 |
| [1500, 2500] | **5（满分）** | 字数合规满分 |
| (2500, 3000] | `5 × (3000 - 字数) / 500` | 线性过渡，保留 1 位小数 |
| > 3000 | 整篇 0 分 | 硬性不合格，所有维度归零 |

计算示例：
- 1200 字 → D1 = 5 × (1200-1000)/500 = 2.0
- 1400 字 → D1 = 5 × (1400-1000)/500 = 4.0
- 2000 字 → D1 = 5（满分）
- 2800 字 → D1 = 5 × (3000-2800)/500 = 2.0
- < 1000 字或 > 3000 字 → 总分直接为 0，不出具维度分

**执行顺序（最高优先级）**：
1. 先统计本章实际字数
2. 若 < 1000 或 > 3000 → **立即终止审核**，输出总分 0，所有维度不评分
3. 若在过渡区间 → 按公式计算 D1，然后继续审核 D2-D9
4. 若在满分区间 → D1 = 5，然后继续审核 D2-D9
```

## 四、对其他 Agent 的影响分析

### 4.1 book_factory.md

**影响**：Step 8c 的通过判定逻辑无需改动（`评分 < 60 → 未通过`），因为整篇 0 分天然满足 `< 60`。但需注意两点：

1. **重写循环行为**（如已实现 bug修复 方案的重写循环）：
   - 字数不合格（0 分）的章节会进入重写循环
   - 重写时 `content_writer` 需要接收到"字数不合格"的信号，确保重写后字数在合规区间
   - **建议**：在 `book_factory` 解析 quality_reviewer 返回值时，识别"硬性不合格"关键词，传递给 content_writer 时增加字数强制约束

2. **日志格式**：字数不合格时日志应区分于普通未通过，例如：
   ```
   | 2026-07-04 12:00:14 | 第1章质检 | quality_reviewer | ... | ❌(0分-字数不达标:850字) |
   ```

### 4.2 content_writer.md

**影响**：当 quality_reviewer 判定字数不合格时，content_writer 在重写模式下需要特别处理：

1. **字数不足（< 1000）**：通常说明 AI 生成中断或输出截断，建议全文重写而非局部修订
2. **字数超标（> 3000）**：需要在重写提示词中增加字数上限强约束

**建议在 content_writer.md 重写模式章节增加**：
```markdown
7. 若质检标注"字数不达标-直接不合格"（总分 0）：
   - 字数不足 → 全文重写，扩展情节填充至 1500-2500 字
   - 字数超标 → 全文重写，压缩冗余描写，控制在 2500 字以内
```

### 4.3 input_monitor.md

**无影响**。input_monitor 监控输入长度增长，与字数合规评分无直接关联。

### 4.4 compliance_tomato.md / compliance_qimao.md

**无影响**。合规检查专注于内容规则（涉政/低俗/未成年等），字数阈值已在 quality_reviewer 中独立处理。

## 六、风险矩阵

| 风险 | 概率 | 影响 | 缓解措施 |
|------|:---:|------|---------|
| 阈值过严导致过多章节被淘汰 | 中 | 写作效率下降，重写次数增加 | 1000 下限已为低标，3000 上限 = 2500 × 1.2 有 20% 容错 |
| AI 生成字数天然波动大 | 中 | 部分优质章节因微幅偏离子数被扣分 | 过渡区间使用连续线性扣分，拒绝一刀切 |
| 整篇 0 分语义过重导致流水线误判 | 低 | 重写循环中可能陷入"写了废 → 废了写"死循环 | 在重写循环中传递"字数不合规"信号，确保重写时调整字数 |
| 平台差异被抹平 | 低 | 七猫（要求 2000-3000）和番茄（要求 1800-2200）使用相同阈值 | 新阈值 [1500,2500] 覆盖两个平台的最佳区间交集；绝对上限 3000 覆盖七猫的 optimal_max |
| 与 book_factory 已有代码的兼容性 | 低 | quality_reviewer 返回 0 分，book_factory 未预期此极端值 | 0 分天然 < 60，触发现有未通过逻辑；建议增加 0 分特殊日志 |

## 七、方案优势总结

| 对比维度 | 旧方案 | 新方案 |
|---------|-------|-------|
| D1 惩罚上限 | 最多 -5（仅清零本维度） | <1000 或 >3000 整篇 0 分 |
| 扣分粒度 | 阶梯式（-3 / -5） | 连续线性（过渡区间） |
| 字数边界 | 依赖平台 `optimal_min`/`optimal_max` 计算浮动范围 | 固定阈值，直观可预期 |
| 过渡区间 | 无 | [1000,1500) 和 (2500,3000] 线性过渡 |
| 极端字数惩罚 | 弱（本维度归零，仍可及格） | 强（整章作废，不可及格） |
| 满分区间 | 依赖平台浮动范围 | 固定 [1500, 2500] |

## 九、附：与已有 bug 修复方案的协同

本方案与 `20260704_quality_reviewer_bug修复.md` 的重写循环方案形成互补：

- **bug 修复方案**解决了「章节不过不重写」的问题
- **本方案**解决了「D1 太容易拿分导致评分虚高」的问题

两者结合后的效果：
1. 字数严重偏差 → 整篇 0 分 → 必然触发重写循环
2. 重写时 content_writer 收到"字数不合格"信号 → 有方向地调整字数
3. 重写后再次质检 → 若字数进入合规区间 → D1 恢复得分
