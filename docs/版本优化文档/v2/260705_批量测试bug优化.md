# 批量测试 bug 优化 — stdin 消费导致的 while 循环提前退出

**日期**: 2026-07-05
**问题**: 执行 `bash scripts/run_pipeline.sh step` 时，只有第一本书（凡人修仙传）跑了 Phase 2 和 Phase 3，其余 4 本书未执行。

---

## 根因分析

### 问题定位：`scripts/run_pipeline.sh:376-390` 的 `while read` 循环

```bash
while IFS='|' read -r name platform track; do       # read 从 stdin (fd 0) 读
    result=$(run_book_pipeline ...)                  # 子 shell 继承 stdin (fd 0)
        └─ run_phase → opencode run ...              # opencode run 也能读到 stdin
done <<< "$books_data"                              # stdin 指向 here-string
```

这是 bash 的经典陷阱：**`$(...)` 子 shell 和其内部启动的进程继承父 shell 的 stdin（fd 0）**，而这里的 stdin 正好是 while 循环用来迭代书籍列表的 here-string。

### 触发链路

```
<<< "$books_data"   →   fd 0   →   ┌─ read (while 循环读取书籍)
                                    └─ opencode run (消费了书籍数据)
```

1. `while read` 从 fd 0 逐行读取 `$books_data`
2. `$(run_book_pipeline ...)` 创建子 shell，子 shell 的 fd 0 指向同一个 here-string
3. 子 shell 内调用的 `opencode run` 可能从 fd 0 读取数据（等待确认、读取终端等）
4. `opencode run` 消费了 here-string 中尚未被 `read` 读取的后续书籍数据
5. `opencode run` 返回后，while 循环的下一次 `read` 读到 EOF，循环提前退出

### 为什么凡人修仙传能跑完 Phase 2/3？

凡人修仙传的 Phase 1 已有 `.phase1_done` marker，Phase 2/3 的 `opencode run` 可能在执行过程中逐步消费 stdin，所以整本书的 Phase 2/3 能跑完，但返回后 while 循环发现 stdin 已空。

---

## Phase 执行模型确认

当前脚本中所有 phase 均为独立 `opencode run` 进程：

| 步骤 | opencode run 次数 | 进程关系 |
|------|-------------------|----------|
| Phase 2.0（初始化+卷纲） | 1 次 | 独立进程 |
| Phase 2.1 ~ 2.N（每章写书） | N 次 | 独立进程 |
| Phase 3（审稿） | 1 次 | 独立进程 |

每次 `run_phase` 拉起全新进程，上一个结束后下一个才启动。**不存在进程复用问题**。

---

## 解决方案

### 核心思路

解耦两条通道：**书籍迭代用 fd 3，opencode run 的 stdin 切到 /dev/null**。

```
改后:
  <<< "$books_data"   →   fd 3   →   read -u 3 (while 循环独享)

                          fd 0   →   opencode run </dev/null (不会消费任何数据)
```

### 具体改动

#### 1. `while read` 改用 fd 3（`scripts/run_pipeline.sh:376`）

```bash
# 改前
while IFS='|' read -r name platform track; do
    ...
done <<< "$books_data"

# 改后
while IFS='|' read -r -u 3 name platform track; do
    ...
done 3<<< "$books_data"
```

#### 2. `run_phase` 中 opencode run 切断 stdin（`scripts/run_pipeline.sh:56`）

```bash
# 改前
opencode run --dangerously-skip-permissions "$@"

# 改后
opencode run --dangerously-skip-permissions "$@" </dev/null
```

### 关于跳过逻辑（marker 文件）

改造不影响已有的跳过逻辑。所有 phase 的跳过条件基于磁盘 marker 文件：

| Phase | 跳过条件 |
|-------|---------|
| Phase 1 | `$book_dir/.phase1_done` 存在 |
| Phase 2 | `$book_dir/.phase2_done` 存在 |
| Phase 3 | `$book_dir/versions/$ver/.phase3_done` 存在 |

凡人修仙传已有 marker 文件，改造后不会重跑。

## 未来扩展：多书并行

如果后续需要多本书并行运行，只需将迭代改为后台 `&` + `wait`：

```bash
for i in "${!BOOK_NAMES[@]}"; do
    run_book_pipeline "${BOOK_NAMES[$i]}" ... &
done
wait
```

因为每本书的 `opencode run` 均有独立 `--dir` 指向各自的 `book_dir`，互不冲突。预期每本书全流程约 40 min，5 本书并行可控制在 ~40 min 完成。
