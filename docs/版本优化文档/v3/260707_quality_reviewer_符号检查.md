# quality_reviewer 对话标点符号检查优化

**日期**: 2026-07-07  
**类型**: 规则增强  
**影响范围**: `quality_reviewer.md`

---

## 背景

中文网文对话必须使用中文双引号 `""`（U+201C / U+201D），但 AI 生成或人工写作中常出现以下错误标点符号：

| 错误类型 | 字符 | Unicode | 示例 |
|---------|------|---------|------|
| ASCII 双引号 | `"` | U+0022 | `"你好"` |
| 英文弯引号 | `""` | U+201C / U+201D | `"你好"` |
| 英文单引号 | `''` | U+2018 / U+2019 | `'你好'` |
| 日式角括号 | `「」` | U+300C / U+300D | `「你好」` |
| 日式白角括号 | `『』` | U+300E / U+300F | `『你好』` |

此前 quality_reviewer 没有任何对话标点检查机制，错误标点可畅通无阻通过质检。

---

## 修改方案

### 1. 新增 G12 强制扫描项（一票否决）

在 §0 强制 Grep 扫描清单中新增 G12，扫描以下字符：

```
"（ASCII 双引号 U+0022）| 「」（日式角括号 U+300C/U+300D）| 『』（日式白角括号 U+300E/U+300F）
```

**阈值**: 0 处（零容忍）  
**命中后果**: 立即终止审核，总分 0，标注 `"对话标点错误-需重写"`

### 2. 执行顺序前置

G12 扫描提升为**审核第一步**，甚至在字数检查之前执行：

> 先运行 G12 扫描对话标点符号 → 命中则立即终止审核，输出总分 0

| 文件位置 | 修改内容 |
