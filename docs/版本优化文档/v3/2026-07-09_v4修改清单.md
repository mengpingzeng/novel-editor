## 二、逐项改动详情

### A. 卷纲补齐 forbidden_patterns 章节

**文件**：`project-agents-template/.opencode/agents/plot_planner.md`

**位置**：§1.2 卷纲结构模板，第 217 行

**改动前**（两行空白占位符）：
```
【本卷受限模式】（从 volume_rhythm_profile.forbidden_patterns 逐条列出，规划时严格规避）
- 禁止：________________
- 禁止：________________
```

**改动后**（强制填表格式）：
```
【本卷受限模式逐条规避举证】（⚠️ 禁止跳过此章节。每条必须填写，不可留空。）
从 volume_rhythm_profile.forbidden_patterns 逐条列出并填写规避方案：

| forbidden_pattern（来源: salt） | 本卷是否存在此风险 | 本卷规避方案 | 验证方式 |
|---|---|---|---|
| {pattern_1} | 是 / 否 | {具体规避措施的章节/段落引用} | {验证方式} |
| ... | | | |
```

**关键设计要点**：
- 将"建议"改为"举证"：不再要求 agent "规避"，而是要求 agent "证明自己规避了"
- 每行必须填写不可留空，若无法规避必须标注"🔴 未完全规避"并说明理由
- "验证方式"列要求引用卷纲内具体章节，防止笼统描述
- 兼容 salt 缺少 forbidden_patterns 字段的兜底逻辑

**预期效果**：
- 卷纲 D4 维度从 8/15 → 13-15/15（+5~7 分）
- 卷纲总分从 87 → 92-94
- 卷纲审核报告最严重单项扣分（-7 分）归零

### B. 总纲禁止清单补齐正反例

**文件**：`.opencode/agents/master_outline_generator.md`

**改动 1**：§3 第 10 节模板重构

改动前：§10 只有 `prohibited_changes[]` 的简单逐条列出格式。

改动后：§10 扩展为三个子节：
- **§10.1** 底层逻辑禁止改动（salt prohibited_changes）——保持原有格式
- **§10.2** 写作禁止清单（新增）——每条必须包含 **[规则] + [量化阈值] + [✅ 正例] + [❌ 反例]** 四部分
- **§10.3** 赛道级禁止模式（salt forbidden_patterns）——保持原有格式

**改动 2**：§五 质量铁律新增两条

新增第 6 条：`§10 正反例强制` — 每条禁止项必须包含正反例对照，正例和反例必须是具体写法示例（≥15 字）
新增第 7 条：`模糊表述量化` — "少""多""合适""自然"等开放式判断词必须附属量化阈值

**关键设计要点**：
- 内置了"少内心独白"和"适配移动端"的量化示例作为模板参考：
  - "少内心独白" → "内心独白占比 ≤15%（约300字/章），以动作和对话为主"
  - "适配移动端" → "段落长度 80-150 字（3-5句），单行不超过 25 字"
- 正反例要求被写入质量铁律（第 6 条），从"建议"升级为"不合格条件"

**预期效果**：
- 总纲 D1 明确性维度从 21/25 → 23-24/25（+2~3 分）
- 总纲总分从 86 → 90-92
- v2 P7/P8 遗留问题一次性关闭

### C. 场景原创度强化

**文件**：`project-agents-template/.opencode/agents/plot_planner.md`

**改动 1**：新增 §0.6 场景骨架原创度检查（插入在 §0.5 L2 替换模式之后）

定义了完整的自查流程：
- **检查触发条件**：公开鉴定/测试试炼/权力羞辱/高辨识度场景等 4 类
- **自查表**：`场景类型 → 常见套路模式 → 本章差异化手段 → 差异化幅度判定`
- **差异幅度三级判定**：高（≥2 个关键节点不同）/ 中（≥2 项显著变化）/ 低（仅改名替换）
- **低差异度的强制补救**：必须触发 ≥1 个 L2 替换方向（环境翻转/附加线索植入/视角切换）
- **参照数据**：优先读取 salt `scene_skeleton_avoid` 黑名单，无则使用通用自查表
- **与 L7 的关联**：直接服务于签约审稿维度 9（原创改写度）

**改动 2**：在 §2.3 批次连续性检查中新增场景骨架原创度批次检查项

```
新增检查项：
- 逐章检查章纲 §10 的"场景骨架原创度自查"结果
- 若本批次 ≥2 章的差异幅度为"低"→ 判定场景同质化风险
- 若本批次所有章的场景类型均为同一大类 → 判定场景类型单一
```

**已知局限**：
- 没有 salt 场景黑名单时，agent 只能在通用层面判断（无法精确对照原著特定场景）
- 效果上限受制于是否同步上线 salt `scene_skeleton_avoid` 字段（方案 ⑫）——本次未含，但 §0.6 已预留接口

**预期效果**：
- L7 签约审稿维度 9（原创改写度）从 11/12 → 12/12（+1 分）
- 每章章纲增加可追溯的场景差异化记录

**改动 2**：写作顺序新增"第〇步：读摘要"
- 在八步写作流程前增加必须步骤——读取伏笔摘要获取伏笔状态、已用拟声词、钩子轮换追踪

#### D-3. quality_reviewer.md

**文件**：`project-agents-template/.opencode/agents/quality_reviewer.md`

**改动 1**：输入约束更新
- 输入 4（纪要）从"最近 3 章"改为"最近 2 章，不超过 2 章"
- 新增输入 4b（必读）：伏笔状态滚动摘要
- 新增 v4 纪要体积铁则：输出不允许超过 15KB

**改动 2**：新增 §4.1 纪要压缩规则
- 满分维度：仅一行 `✅ D{X} {得分}/{满分}`（~50B）
- 轻微扣分（≤1 分）：两行简要记录
- 显著扣分（≥2 分）：展开分析 + 修改建议
- 禁止满分维度输出详细分析段落

#### D-4. 新增模板文件

**文件**：`project-agents-template/伏笔状态滚动摘要-template.md`

包含 4 个追踪表：
1. **伏笔状态汇总**：伏笔 ID / 级别 / 状态 / 预期回收章 / 最后推进章
2. **已使用拟声词黑名单**（最近 5 章）：避免 writer 重复使用
3. **已使用重复短语黑名单**（最近 5 章）：避免跨章句式雷同
4. **钩子类型轮换追踪**（最近 5 章）：确保钩子类型轮换合规

**设计原则**：
- 替代"全量历史纪要拼接"，将跨章上下文压缩为一张轻量表（~2-3KB）
- 每章仅追加一行（伏笔状态）或更新对应行，体积不随章节数增长
- 解决了滑动窗口的核心矛盾：不读历史纪要但需要知道跨章伏笔状态

**预期效果**：
- Ch5 正文撰写输入从 145KB → 预计 ~60KB（-58%）
- Ch5 纪要生成输入从 138KB → 预计 ~50KB（-64%）
- 输入监控评分从 35 → 70-80（+35~45 分）
- 输入膨胀率从 +341% 降至 +50% 以内
