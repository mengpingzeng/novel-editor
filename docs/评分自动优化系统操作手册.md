# 自动化评分系统操作手册

> 当前版本：V2 | 更新日期：2026-07-04 | 平台：OpenCode

---

## 零、快速开始（5 分钟上手）

### 0.1 确认原著文件已就位

```
workspace/repo/{原著名}/source.txt
```

例如：
```
workspace/repo/凡人修仙传/source.txt
workspace/repo/斗破苍穹/source.txt
```

### 0.2 先跑 dryrun 验证系统

```bash
bash scripts/run_pipeline.sh dryrun
```

这会用「甄嬛传」写 3 章完整验证全流程（框架 → 写书 → 审稿），大约需要 10-30 分钟。产物落在 `workspace/_dryrun/`，不污染正式目录。

### 0.3 dryrun 通过后，配置正式任务

编辑 `workspace/iteration-state.json`：

```json
{
  "target_chapters": 20,
  "active_books": [
    {"name": "凡人修仙传", "platform": "番茄小说", "track": "仙侠"},
    {"name": "斗破苍穹", "platform": "番茄小说", "track": "玄幻"}
  ]
}
```

### 0.4 启动正式执行

```bash
bash scripts/run_pipeline.sh step
```

系统会逐本处理：**书A Phase1→2→3 → 书B Phase1→2→3 → ... → 跨书总结**。每本书之间完全隔离，互不影响。一本书跑完才跑下一本。

## 一、脚本说明

脚本位于 `scripts/run_pipeline.sh`，封装了 V2 三层架构的完整流程。

### 命令

```bash
bash scripts/run_pipeline.sh dryrun    # 甄嬛传 × 3章 全流程自测
bash scripts/run_pipeline.sh step      # 全部 active_books 全流程
```

### step 模式的执行流

```
┌─ 书A ─────────────────────────────────────┐
│ Phase 1: 框架生成 (白皮书+赛道+门面+盐值+总纲) │
│ Phase 2: 写书 (chapters 章)                 │
│ Phase 3: 审稿 (全层级评分+签约审稿+流程审计)   │
└───────────────────────────────────────────┘
┌─ 书B ─────────────────────────────────────┐
│ Phase 1: 框架生成                           │
│ Phase 2: 写书                               │
│ Phase 3: 审稿                               │
└───────────────────────────────────────────┘
...
跨书总结
```

### 中断与重跑

- **中断后重跑**：直接再执行相同命令，脚本会自动跳过已有 `.phase*_done` 标记的步骤
- **强制重跑某本书的某阶段**：删除对应标记文件（如 `workspace/books/凡人修仙传/.phase2_done`），再执行脚本
- **不限时间**：无超时限制，一本书可跑数小时甚至数十小时

### 查看进度

每本书的完成标记：
```
workspace/books/{书名}/.phase1_done   ← Phase 1(框架) 完成
workspace/books/{书名}/.phase2_done   ← Phase 2(写书) 完成
workspace/books/{书名}/versions/{v}/.phase3_done  ← Phase 3(审稿) 完成
```

执行日志：
```
workspace/books/{书名}/versions/{v}/自动化处理日志.md
```

## 四、V1 → V2 核心变化

| 项目 | V1 | V2 |
|------|----|----|
| 执行方式 | `/iterate step` 一键 | `bash scripts/run_pipeline.sh step` 一键 |
| 架构 | book_factory 单体 | 三层解耦（框架/写书/审稿） |
| 每本书执行流 | 所有书先 Phase1，再全部 Phase2，再全部 Phase3 | 书A Phase1→2→3 完成后再书B |
| 写书入口 | `@book_factory` | `chief_editor` |
| 审稿入口 | `@signing_reviewer` | `reviewer_orchestrator`（含 8 个评分 agent） |
| 质检 | 评分 <60 跳过 | 最多 3 轮重写，取最高分 |
| 字数约束 | 最多扣 5 分 | <1000 或 >3000 字 → 整篇 0 分 |
| 流程审计 | 无 | pipeline_auditor 四维 100 分制 |
| 日志文件 | execution-log.md | 自动化处理日志.md |
| novel_metadata | books/{书名}/ | versions/{v}/发布/ |

## 五、V1 历史内容（仅供参考）

> 以下为 V1 版本的内容，V2 已废弃 `book_factory` 和 `signing_reviewer` 单独调用的方式。

### 5.1 What？这是什么系统

这系统帮你做两件事：

1. **自动写书**：给定一本原著小说，AI 自动完成「拆解原著 → 赛道映射 → 生成书名简介 → 逐章写正文 → 质检合格 → 输出完整小说」
2. **严格打分**：对写出来的小说，AI 用编辑视角严格打分（60 分及格但不放水），告诉你哪里写得不好

写书的不认识做评分的，评分的也不关心这书谁写的——两个系统完全独立。

### 5.2 准备工作

确认原著 txt 在 `workspace/repo/{原著名}/source.txt`。

编辑 `workspace/iteration-state.json`：

```json
{
  "target_chapters": 20,
  "active_books": [
    {"name": "凡人修仙传", "platform": "番茄小说", "track": "仙侠"}
  ]
}
```

### 5.3 V1 命令（已废弃）

| 命令 | 说明 |
|------|------|
| `/iterate dryrun` | V1 自测 |
| `/iterate step` | V1 一键跑完（不可用，V2 架构已变更） |
| `@book_factory source_name=... target_platform=... chapter_count=... version=...` | V1 单本写作（已废弃） |

请使用 V2 命令：`bash scripts/run_pipeline.sh dryrun` 或 `bash scripts/run_pipeline.sh step`。
